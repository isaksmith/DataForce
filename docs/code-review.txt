DataForce - Code Review and Hackathon Submission Summary
========================================================

Project
-------
DataForce is a unified digital friction intelligence platform built for Hack The Plains 2026.
It analyzes end-to-end banking interaction data (sessions, errors, support, feature costs,
customer profiles, branch context) to identify high-friction customer journeys and prioritize
interventions that reduce support cost, improve resolution rates, and lower churn risk.

Core objective:
Move the organization from passive error logging to active, cost-aware intervention.

Business framing:
- Current support call estimated cost: $12.00
- Digital/self-serve intervention target cost: $0.01
- Product goal: detect friction early, guide recovery, and escalate only when needed


Judging-Oriented Executive Summary
----------------------------------
This project demonstrates:
1) Data integration across six source datasets with consistent normalization and profiling.
2) A friction scoring methodology tied to both experience outcomes and financial impact.
3) A clear prioritization framework (Feature Friction Matrix + ranked intervention targets).
4) Explainable visual analytics across operational, customer, and support dimensions.
5) An extensible simulation layer backed by generated customer personas.
6) Production hardening work for Streamlit Cloud deployment stability.

Application Scope (Final Delivered Pages)
------------------------------------------
1) Home (DataForce.py)
- National command-center view with branch choropleth map and telemetry cards.
- High-level operational signal: failed transactions, support load, error pressure.
- Terminal-inspired UI for pitch/demo readability.

2) Explorer
- Generic dataset browser for all source tables.
- Inline filtering + row/column metrics + table preview + chart by dataset type.

3) Branches
- Geographic branch distribution analysis by state and city.

4) Customers
- Segment/churn/tenure exploration with demographic and branch context views.

5) Digital Sessions
- Session outcomes, channel and feature usage, top errors.
- Sankey flow (Channel -> Feature -> Outcome) to reveal drop-off and failure paths.

6) Error Codes
- Error impact summary with session linkage, top feature attribution.
- Treemap and bubble chart views for frequency and impact.
- Label cleanup from NA to No Error for interpretability.

7) Feature Costs
- Success vs failure cost comparison per feature.
- Directly powers friction cost premium weighting.

8) Support Interactions
- Interaction type, reason-code, and resolution mix.
- Feeds 24h/72h support follow-up signals into friction scoring.

R/Shiny visualization blueprints are included directly in:
- Branches
- Customers
- Digital Sessions
- Error Codes
- Feature Costs
- Support Interactions

Blueprint intent:
- Provide a direct Python-to-R implementation pathway for each core visualization page.
- Enable teams with R-first analytics stacks to reproduce equivalent dashboards quickly.

9) Friction Score
- KPI panel + Feature Friction Matrix + priority ranking cards.
- Quantile-based priority zone for intervention targeting.
- Central decision-support page for product and operations.

10) Knowledge Graph
- Interactive schema and relationship view across core datasets.

11) Customer Simulations - In Progress
- Persona-driven simulation controls and visual outputs.
- Persona generation references MiroFish OSS lineage:
  https://github.com/666ghj/MiroFish
- Cloud-safe fallback uses a representative 5,000-persona sample.

12) About
- Consolidated project narrative (vision, components, page map, datasets, stack).
- Updated for judging clarity and explicit MiroFish backend reference.

Deployment Issues Encountered and Resolutions
---------------------------------------------
Issue A - Streamlit Cloud stalling on startup
Likely cause:
- Blocking remote GeoJSON retrieval from GitHub at page render time.
Resolution:
- Added fetch timeout and cache layer for GeoJSON retrieval.
- Removed unsafe caching of mutable Folium map object that broke rendering.

Issue B - Full-app crash risk from optional simulation dependency
Likely cause:
- mesa import failure propagating through shared utility imports.
Resolution:
- Made Mesa import optional in shared utils.
- Added runtime guard and user-facing error path in simulation execution.

Issue C - Persona artifact missing in cloud
Likely cause:
- Full persona file too large for GitHub limits (>100MB).
Resolution:
- Added stratified 5,000-row sample artifact for cloud deployment.
- Implemented full-file preferred, sample fallback behavior in simulation page.

Issue D - Sidebar/nav clarity
Resolution:
- Explicit page ordering in pages.toml.
- Added About page and adjusted placement under Customer Simulations.
- Marked simulation page as in-progress in title.


About Page Context (Integrated)
-------------------------------
Vision:
- Failed/abandoned sessions create direct support cost, churn risk, and trust erosion.
- DataForce detects friction early and routes users to the lowest-cost successful resolution.

Core components (final state):
1) Friction Engine
- Composite score built from session outcomes, retries, error codes,
  feature failure premiums, and support follow-up signals.

2) Proactive Intervention Layer
- Contextual guidance and targeted recovery paths based on customer profile
  and friction context, modeled and stress-tested in simulation workflows.

Reference integration:
- Persona backend concept linked to MiroFish open-source framework.
